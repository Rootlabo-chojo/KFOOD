# 📖 칸반보드 업무 프로세스

1️⃣ 칸반보드 단계별 역할 및 규칙

| 단계                 | 핵심 역할                              | 주요 규칙 및 행동                                                                              |
| ------------------ | ---------------------------------- | --------------------------------------------------------------------------------------- |
| Quick Task Pending | 단기 업무 대기 (업무지원 등 즉시 처리 가능한 일)      | 담당자는 본인의 업무 상황을 고려하여 해당 열에서 직접 테스크를 가져와 'In Progress'로 이동시킵니다.                          |
| Project Pending    | 장기 프로젝트 대기 (기획, 개발 등 여러 단계가 필요한 일) | 프로젝트 리더(PL) 또는 팀장의 지시에 따라 우선순위가 높은 테스크부터 'In Progress'로 이동시킵니다.                         |
| In Progress        | 진행 중 (담당자가 실제 업무를 수행하는 단계)         | 현재 내가 진행하고 있는 모든 업무는 반드시 이 열에 있어야 합니다.                                                  |
| Approval           | 승인 대기 (완료 전 최종 확인을 기다리는 단계)        | 승인 요청 시, 코멘트로 @승인자를 태그하여 알림을 보내야 합니다. 승인자는 24시간 내 피드백을 원칙으로 합니다.                        |
| Complete           | 완료 (모든 과정이 최종 종료된 상태)              | 'Approval' 단계에서 최종 승인된 테스크만 이곳으로 이동할 수 있습니다.                                            |
| Revision           | 수정 요청 (승인 과정에서 반려된 상태)             | 승인자는 반려 사유를 코멘트에 명확히 기재해야 합니다. 담당자는 코멘트 확인 후, 테스크를 다시 'In Progress'로 이동시켜 수정 작업을 시작합니다. |
